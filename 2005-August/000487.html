Well the reason is simple: It's no Qt project. NoteEdit was
in history a Qt-only project and inbetween both KDE and Qt could
be used. Currently it requires KDE though. KDE has it's own way
to create menus in a way not compatible with the designer. I don't
know but maybe it's possible to edit the menu graphically with
kdevelop.

2.8.0 does not crash for me. There are some possibilities
that could lead to a crash: Newer gcc compiler in combination
with an older unpatched tse3 library. Also some midi kernel
modules need to be loaded or (I did not test this personally)
maybe it's enough to start NoteEdit without tse3.
