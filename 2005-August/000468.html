My reorganizing steps are very easy.


I try to remove as many methods from NMusElement as possible by
moving them into other classes where they logically belongs (the context
helps a lot). 

After making it, I'll reach the base class. A small class in which every
method is understandable. The other classes can be built onto this class
gradually.

The heart of a code is placed in the headers. So if the headers are
reorganized, the code will also be reorganized.

Awful implementation is not as big error as awful headers.
Implementation can be changed any time without much pain.

Telling the truth, in the Krusader project I made code reorganizing.
I changed only the headers, and the implementation followed it. Big
work, but worths the effort.
