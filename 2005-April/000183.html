Hi,

While working on joining Staff properties and the voice
dialog I noticed some very unusual behaviour of noteedit:

If you press "Apply" the properties are globally saved.
If you press "Cancel" afterwards the previous properties are
restored.

All the applications I know of have the following behaviour:
If you press "Apply" the properties are globally saved.
Pressing Cancel does not restore the previous values.
Several programms additionally
- Disable Apply/Cancel if no property was changed
- Disable Apply/Cancel if apply was pressed.

Do you know if this is a usual noteedit behaviour
and why this is done this way ?
