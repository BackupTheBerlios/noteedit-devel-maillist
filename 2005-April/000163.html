Now that's interesting. In my Makefile LEX / YACC is defined as
LEX = flex -l
LEXLIB = -lfl
YACC = bison -y -d

So you are right, lexik.ll is generated by flex. The interesting
thing is how the cc files are created out of the ll files. The
Makefile.am contains some lines including .ll.cc / .yy.cc directives
(i.e. if the makefiles encounters a .ll / .yy file, the corresponding
.cc file is creating by using the commands found in the directive.
Now your problem might have two solutions:

1) Remove the directives from the Makefile.am (they are generated
with newer automake anyway automagically).

2) Remove the automatically generated directives from the Makefile.
Even in my Makefile both directives are present. Not something which
I particulary like. I'll do some test of this on my system as well.
