> Hi,
> 
> While working on joining Staff properties and the voice
> dialog I noticed some very unusual behaviour of noteedit:
> 
> If you press "Apply" the properties are globally saved.
> If you press "Cancel" afterwards the previous properties are
> restored.
> 

I took a deeper thought into this. I guess it would be good
to test the values on the screen for the user before really
applying it. There are two possibilities for that:

- Save every value twice in the staffPropFrm class
- When changing a value show the changes at once in the
   mainframe widget.

I tend to do the second as you can still Cancel your complete
action and see the effects of your change.

In the other case I would have to add a "Test" button so it is
possible to test the values before applying them. Maybe it is
even better to rename the "Apply" button to "Test".

What do you think?
