noteedit-devel-admin at berlios.de schrieb:
> Yes, I think we should release 2.8.0 version and split the 
> stable/unstable branches immediately. I have a question about Qt though: 
> Are there any big API differences between Qt3.x and Qt4.x 
> APIs/libraries? Because Qt4.0 has a release date in June afaik.

Indeed there are big differences between Qt4.0 and the current one.
They build in a compatibility layer so the developers can slowly
switch to the new Qt4.0 API. Also the designer has changed a lot
in it's functionality (as I read last there was no longer a possibility
to code with the designer). They have split up the library. Many
of the classes which were not accessible without subclassing are
no accessible, OTOH several classes were removed AFAIK.

I'll try to get a beta release ready at the weekend so we get
the release out quite soon. I'll set the version on the corresponding
files if no one has time this week to do this.

Concerning the split: You should check out the 2.8.0 branch as soon
as I created it. The "HEAD" branch will then be the 2.9.0. I'll
give some instructions as soon as I have done it.
