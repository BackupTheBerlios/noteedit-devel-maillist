Hi Georg,

noteedit-devel-admin at berlios.de schrieb:
> Is there a way to make this configurable? I would find it sad if the
> integration into kde would be removed. I tend to respect the work the
> authers of noteedit did ...

Theoretically we could maintain both a Qt and KDE Version of NoteEdit
but I think it would mean quite some work to keep both versions as
not only for the main window and the menu there has to be code doubled
but also for file dialogs, settings, standard dialogs etc.
Of course I do respect the works of the original noteedit author(s)
as well. Going to pure Qt will make things a little bit more 
complicated. OTOH it is much easier to port to other platforms after
this step as we are only dependent on one user interface API.

> Persoonally I don't need the integration, but I just imagine the shuffle
> of files on the Desktop to call noteedit might be easier. More
> substantial critic I can't give in the moment. The option to be able to
> run without kde is a plus, of course.

Concerning the KDE integration maybe we could support a minimal
desktop integration (mainly for theming and such). It would have
to be done in a way that these addons can be switched easily with
#ifdef KDE. Other things like Midi / TSE3 / arts (the audio integration)
is a completely different topic I would not change at this time.
