Hi,

I developed a design proposal for the staff properties
(new with voices included). I've included it into the
attachment (use the qt3 (3.3.4) designer to view it).

Due to place restrictions the selection of the staff
name has to be done via combobox instead of tabs. Even
if we do not join staff properties and voice dialog it
would be a good idea to be able to select a different
staff via combobox.

The staff property dialog has a wrong order of the
buttons below. I corrected this as well as the buttons
are no longer on the same height as staff itself.

I don't like the position of the create voice button.
Maybe someone has a better idea how to create a voice.

I also removed the help button as it does not show any
context sensitive help for the staff properties. It's
better to create real context sensitive help with a '?'
icon (using the Whats this functions).
