Ugh. I hate this kind of bugs, mostly because of weak NoteEdit structure 
:(. I didn't dig into the bug though, but I persume that NoteEdit 
doesn't have a function that would return you the notes that are in 
timing above the current, in other voice (chord). You could then seek 
the previous element of every first-voice note (above the current one) 
and if this element is a measureline or similar, then the measureline 
should be before the current note as well.
The other way I'd prefer more, is that measurelines of any kind should 
be written in non-first voices as well. This would easily resolve RFE (I 
don't remember the number), which suggested writing measurelines in 
non-first voices as well.
All in all, everything above should require a thought or two thoughts 
more:). I hope you have some better ideas than I do in the moment!
