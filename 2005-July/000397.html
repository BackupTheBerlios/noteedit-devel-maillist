Hm... You're thinking something about a OOo module for editing notes?
I've been thinking about it myself a few months ago and I came to the
conclusion that OOo is actually a publishing software (something more
like LaTeX or LilyPond for notes). I think the nicest and the easiest
way for now would be a cooperation with Lily, something like:
1) export to .ly
2) convert to the latest version of syntax (convert-ly filename.ly)
3) compile the file to .svg vector format (lilypond --svg --no-ps
- --no-pdf --no-png filename.ly)
4) import the .svg to OOo

Now, when you'd double click on the score graphics in OOo, it could
run NoteEdit with the appropriate .not file. The idea is to have both
.svg file for the real graphics and always a .not file for the ability
to edit it. The downside is that you need LilyPond beside NoteEdit and
that there could be slight differences from time to time, because
LilyPond uses a much more sophisticated engine for rendering the score
than NoteEdit.
