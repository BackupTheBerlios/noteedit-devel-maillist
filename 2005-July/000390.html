> 2) I played downloaded&compiled&installed Qt4.0. I like it so far, but
> wasn't able to make much with NoteEdit yet (configure doesn't find
> it?!?). Can you say anything about your plans, when and how are you
> going to migrate to Qt4.0? We'd have to update our configure to work
> properly (or maybe even rewrite it completely?), remove code in #if
> QT_VERSION <300 block, make changes to fit a new Qt API and announce
> that NoteEdit 2.9 requires now Qt >= 300.

I haven't thought about it and I probably won't start thinking about
it until Noteedit 2.8.1 has been released. Scripting and the printing
dialog are the main features needed by NoteEdit in the near future IMO.
I want to wait for that until my distribution officially includes QT4
in some form (that will probably happen in late autumn, end of october
or so). Basically I would drop Qt 3.x support completely as there is
a QT3.x backward compatibility interface in Qt4 and because the 
differences are to large for maintaining backward compatibility. I
also don't think we will se an unstable release this year except we
some developers having more time than me to work on it.

> 3) I tried compiling NoteEdit with gcc-4.0, but had no luck. I fixed
> the configure script and it works fine. The problem is that it cannot
> compile TSE3 headers (MANY syntax errors like the wrong use of
> type-def, friend classes, templates etc.!). I contacted Pete and I
> hope he'll manage to release a gcc4-compatible version asap. I don't
> think Debian is going to accept our program without being gcc4-compatible.

gcc4.0 is in Cooker for Mandriva and I won't switch to it until the
next version of Mandriva is released (maybe if I have time I'll test
the release candidate for Mandriva somewhen in September/October).
I think all other distribution will switch to gcc4 before Debian
will officially start to use it (i.e. at least in testing).

I hoped to manage more than the homepage update an the release

I'm very satisfied with the work and the plans you do and hope
