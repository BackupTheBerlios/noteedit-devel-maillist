noteedit-devel-admin at berlios.de wrote:

> Hi! I've some questions...
>
> 1. What is the main reason that the user cannot edit the bar lines
> directly?
>
> I've found a short key '|', but it doesn't really work. Buggy.
>
> I have scores which has no specific time signature, it changes from
> bar to bar.
>
> As bar lines can be deleted, but one cannot create new one, this
> makes hard to correct it.

For placing a barline, the TAB key is used. For repeat barlines, one
of the three kinds in the toolbar has to be selected and then placed
in the score. For the final barline, an option from menu has to be
chosen. For setting a N-time repeat, you first have to place a repeat
bar and then Edit->Set N-time repeat.

Using a pipe (|) key is IMO not needed anymore and will be removed in
future. About the non-metrum music, I commited a patch that doesn't do
automatic placement of barlines and similar if no metrum is defined.
It should work properly as I tested.

> 2. Does NoteEdit support software MIDI sequencers, such as
> timidity?
>
> I've installed linux onto 7-8 different computers, but the hardware
> MIDI sequencer only worked on 2 of them.
>
> Was it my fault, or just ALSA doesn't support most of the MIDI
> sequencers?
>
> If I start timidity as a background process, it works, but most of
> the users has no idea how to do it...

I myself run Timidity in background (I have a cheap integrated VIA
sound chipset on my laptop). Under Debian, all I had to do is to
install timidity and commented out ENABLE=1 line in
/etc/init.d/timidity so it starts as a server on startup every time I
turn the computer on. I think there's not much NoteEdit can do here.
It could run the timidity server when NoteEdit starts automatically,
but it couldn't run it in realtime high-priority mode as it's not run
by root (the startup script is).

> 3. If I use 'keyboard mode' and press ENTER, nothing happens. The
> documentation says that a note should be inserted...

Hm... it works for me. I simply activate the Insert from Keyboard mode
(a big A letter on a button on the blue backroung without a glue) and
then I use up/down arrows for selecting the pitch and ENTER to place
the actual note.

> 4. The page settings for export are not saved in the .not file. The
> user has to fill the dialog up every time, he/she restarts
> noteedit.

True. That's one thing that needs to be included in the new NoteEdit
file format. The problem at the moment is that NoteEdit actually
cannot store any special values than most of the current ones as the
.not file is MUP-compatible and doesn't offer all the features
NoteEdit does.
