Thank you for the answer. Yes, printing is really the most important 
thing to do. To tell the truth, printing and exporting into OOo should 
use the same code.

The OpenOffice.org 2.0 Draw file format is a zip file containing XML files.

These XML files contain lines, texts, bitmaps,...
It's similar to the windows metafile (WMF).

All I have to do is to create a new Paint Engine, which collects the
drawing shapes from QPainter and stores it in odg (The OpenOffice 2.0
Draw format). It's easier, than I expected.

Drawbacks of my solution:
- QPaintEngine is defined in QT 4.0, which requires at least KDE 4
- The odg file format requires at least OpenOffice 2.0

I don't suggest to keep the compatibility with KDE 3.x, and OOo 1.x, as 
it makes the solution much harder. Conditional compilation can be used, 
till everybody will use at least KDE 4.x.

If it is possible, it would be nice if noteedit could print a page into 
a bitmap, or png or anything else... That can also be inserted into OOo 1.x.
