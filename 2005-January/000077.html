Hi Michael,

Welcome to the club ;-) I've added you as a developer to the project.
We're close to release 2.7.4 though I have not much time currently
to help on that (no problem to create tarballs for me). The main
problems I encountered are project management with KDE.

When you have checked out the current SVN code you could either
try to use the configure scripts inclued in the 2.7.3 archive or

- Create a new "noteedit" project with kdevelop
- Copy the SVN directory (I call it noteedit_work, a working copy)
- Copy the admin files and the Makefile.cvs to the working copy.
- Try to build it: make -f Makefile.cvs, ./configure.
- Check if Makefiles were created for doc and noteedit directory.
   if not add some line to configure.in:

AC_CONFIG_FILES([ doc/Makefile ])
AC_CONFIG_FILES([ doc/de/Makefile ])
AC_CONFIG_FILES([ doc/noteedit/Makefile ])
AC_CONFIG_FILES([ kguitar_excerpt/Makefile ])
AC_CONFIG_FILES([ noteedit/Makefile ])
AC_CONFIG_FILES([ noteedit/icons/Makefile ])
AC_CONFIG_FILES([ noteedit/resources/Makefile ])
AC_CONFIG_FILES([ po/Makefile ])

These have to be added after the line
AC_CONFIG_FILES([ Makefile ])

Try to 'make' it.

QT: The user interface is created from XML files
uic take a .ui files and generated .cpp / .h files.

KDE: Don't have much experience with KDE development myself.

If you have a good idea for solving the build nightmare ;-)
just drop us a note on the devel ML.

You can also try to use kdevelop as integrated development
environment. It has gdb support.
