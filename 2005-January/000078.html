> yesterday I compiled both vkeybd and kaconnect without problems. Apparently 
> there is an issue with my ALSA setup, because both fail with the message 
> "ALSA lib seq_hw.c:446:(snd_seq_hw_open) open /dev/snd/seq failed: No such 
> file or directory". For the time being I don't know how to solve that.
> 
> Won't be able to make much progress on bug #3088 ...

This can be easily solved. You need to modprobe snd-seq drivers like
snd-seq
snd-seq-dummy
snd-seq-midi-emul
snd-seq-virmidi      --> Virtual Midi in case you don't have "real" Midi
snd-seq-instr
snd-seq-midi-event
snd-seq-device
snd-seq-midi

vkeybd works fine with that.

You might want to add some of these to /etc/modules (2.4.x) or 
/etc/modprobe.preload (2.6.x).

> In case you are familiar with ALSA: my setup is Fedora Core 3, on-board AC97 
> audio. /dev/snd only contains control, pcm and timer devices.

I've installed Fedora Core 3 today and want to test the KDE installation 
as well as my external USB sound card (24 Bit 96 Khz) with Noteedit.

You can also try the sound server jack(it) as alternative to arts or 
esd. Take a look at qjackctl.sf.net. You might want to have the 
realtime-lsm module installed, see www.joq,us "Linux Audio kernel modules".
