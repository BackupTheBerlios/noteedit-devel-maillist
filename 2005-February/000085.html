A slightly different proposal, based on the Linux kernel versions:
use x.y.z with odd y's for development and with even y's for stable.

I believe this is a common numbering scheme, e.g. LilyPond also uses it. This 
would imply we need to copy 2.7.4 to a stable 2.8 release (with bug fixes in 
releases 2.8.1, 2.8.2 etc.).

New development would go into 2.9 (and, when stable, become 3.0).

Regards, Leon.
