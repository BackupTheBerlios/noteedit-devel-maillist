Hi all,

midi import on my x68-64 box crashed. After a hunt, the attached patch 
will do the job for me again, so a real fix won't be too far :)

lastPlayableIdx seems to be a flag if lastPlayable->getMidiLength() is 
initialized or not. This bug might also have hit 32-bit builds, I guess. 
I created bug #3483.

Importing midi files from the internet or other programs is very 
valuable. We should increase the number of accepted tracks, find out how 
to quantize and document that more obvious.

In case you want to try it, here's how I do midi import:
    file -> new
    tse3 -> create tse3 song
    tse3 -> import midi, select a midi file
    tse3 -> tse3 to score
   
Or is there an easier way? Anyway, now it's even more fun.
