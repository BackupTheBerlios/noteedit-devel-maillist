Hi all,

from Stephan Binner I got this ./configure report:

checking for strip... (cached) strip
checking for Cygwin environment... (cached) no
checking for mingw32 environment... (cached) no
/bin/sh: admin/ltconfig: No such file or directory
configure: error: libtool configure failed
make: *** [configure-work/noteedit-2.7.4/configure] Error 1

I found out that in the admin directory some soft links where missing. Okay, I substituted already the tarball on the website with what I made now with this recipie:


B = noteedit-2.7.4
A = noteedit-2.7.4_beta1

build_tar_ball:
    rm -rf $B
    svn export svn+ssh://svn.berlios.de/svnroot/repos/noteedit $B
    cp $A/configure.in    $B
    cp $A/Makefile.in     $B
    cp $A/acinclude.m4    $B
    cp $A/aclocal.m4      $B
    cp $A/configure       $B
    rm -f $B/Makefile.cvs
    cd $B/admin ; ln -s old-libtool.m4.in libtool.m4.in ; \
                ln -s old-ltcf-c.sh ltcf-c.sh  ; \
                ln -s old-ltcf-cxx.sh ltcf-cxx.sh  ; \
                ln -s old-ltconfig ltconfig  ; \
                ln -s old-ltmain.sh ltmain.sh
    tar cvf $B.tar $B
    gzip $B.tar


I hope not too many people got stucked with this. In the Suse 9.2 rpmbuild/spec build process these links seem not to be needed.

Best Regards,
Georg.
