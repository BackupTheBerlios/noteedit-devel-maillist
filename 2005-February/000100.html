Hi Reinhard,

no offense taken :-).

Your remark about the value 12 is correct, a better solution would be to end 
the string table with a null. The way it is coded now has implicit knowledge 
about the table length in resource.cpp encoded in uiconnect.cpp which, of 
course, is a maintenance problem.

However, the same construction is used in more places and for a "last minute" 
fix I did not want to deviate from the existing coding style or rework it 
completely. Improving the coding style is something I prefer to do only 
during development, not in beta. For 2.7.4 we should aim for safety and fix 
only killing bugs using the smallest possible code change.

Regards, Leon.
