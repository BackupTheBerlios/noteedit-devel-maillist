Ok, I have found a python <-> ruby comparison on the ruby page (FAQ):
http://www.rubygarden.org/faq/entry/show/14

And a comparison chart:
http://www.approximity.com/ruby/Comparison_rb_st_m_java.html

I cannot recommand perl as it's syntax tends programmers to make
unreadable code. It's of course very powerful and has a lot of
add-on modules (CPan) but it's no pure object-oriented language like 
ruby (python has also procedural elements like Perl). I don't know
LUA but from it's description it's also a procedural language. The
main difference between Python and Ruby seems to be the complexity.
You can also download a ruby book:

The basic question is if we want to have every element of Noteedit
as an object or not. I vote for yes as it is the more natural way
of programming. It also should make the transition from C++ NoteEdit
code to scripting code more easily.
