Hi all,

Please find attached corrections from Jens Seidel for the German
translation of 2.7.4, along with english typos fixes. This patch was
sent to the Debian Bug Tracking System (see also [1] and [2])
http://bugs.debian.org/313942 and http://bugs.debian.org/noteedit)

Unfortunately I haven't had the time to test 2.8.0-rc1 yet, but I have
prepared a debian package for 2.7.4. The current version can be found at
http://piem.org/debian/noteedit/ . You will find all the modifications
made in the .diff.gz file. Comments and feedback welcome.

I noted the following issues in version 2.7.4:

 - i did get to reproduce #230891 after loading a large
   multichannel midifile [1].

 - the kprinter box is broken, only pdf works. it seems to fail finding
   tools in /usr/bin. the error is: "Cannot save print file to
   /home/piem/print.ps. Check that you have write access to it."

 - tse3play does not seem to be willing to play at all (most likely a
   tse3play bug).

 - autobar, on some files [2], eats all the cpu and never returns

 - autobeam sometimes crashes on createUndo

 - quite a few warnings when compiling (debian unstable / powerpc)
   examples include:

mainframewidget.cpp:674: warning: `connectItem' is deprecated (declared at 
   /usr/include/kde/kaccel.h:337)
mainframewidget.cpp:675: warning: `insertItem' is deprecated (declared at 
   /usr/include/kde/kaccel.h:329)

...

tse3handler.cpp:220: warning: comparison is always true due to limited range of 
   data type
tse3handler.cpp:223: warning: passing `double' for converting 1 of `
   TSE3::Clock::Clock(int)'

 - temporary files locations are a bit weird (should use kde temp dir
   instead) 


Cheers, Paul
