Hi,

I looked at both the old print support and the goals I have
with printing myself. As a first step I decided to use the
KDE printing support already integrated into NoteEdit and
only expand it so it shows the selected music system print
options instead of only abcmusic options.

Though there is a preview integrated into this dialog I
decided to keep the direct print preview menu entry as
it is more convenient for the user to use that dialog.
Also I will keep the settings for the print preview as
described on the NoteEdit wiki.

Later if we migrate to Qt4 we need to replace the
print dialog itself as well as several other KDE dialogs.

Status:

- You still have to enable printing with --with-printing
to test the current printing and later the enhanced
printing. As soon as a certain stability is reached I
will try to enable printing by default.
- Nothing in SVN yet (only tested the settings UI shortly)
- Settings UI is finished excepts labels and some layout changes
- menu entries done
- saving settings done
- default settings done
- resources for printing options done
- started expanding print class / methode

ToDo:
- finish print class / methods
- finish Settings UI

Also some other things I noted:
- Options are only saved when the NResource obect is
   destroyed (i.e. apply does not save the settings)
- NResources are unstructered
- direct access to NResource variables
