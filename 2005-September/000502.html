Hi Csaba!

My vote gets the complete name changes throughout the code as well. I
would advise you though, to copy the current /trunk branch to the
separate one, eg. /branches/csaba-coderewrite2. This way, if any
show-stopper bugs occur in your work, you won't influence on the trunk
branch. But when your code is mature enough, you can simply merge your
branch with the trunk one. I think there won't be any major changes and
implementations in this and the following month around status and
similar internal structures. However, if any minor features get
implemented, which use status in any way, they can get quickly fixed on
the merge.

>I had the following problems:
>
>- the name 'status_' should be changed to 'property_' as
>  status_ was misleading
>- for better understandability in each local variable 'status' should
>  be changed to 'property'

>- constants beginning with STAT_ should be changed to PROP_

>instability. More than 1000 lines should be changed.
