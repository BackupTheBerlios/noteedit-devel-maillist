I only unified the status_ and status2_ variables.

The setter / getter methods were not made.

I had the following problems:

- the name 'status_' should be changed to 'property_' as
  status_ was misleading
- for better understandability in each local variable 'status' should
  be changed to 'property'
- the 'status' member of NNote should also be changed to 'property'
- some structure names contain 'status' and this requires also change
- constants beginning with STAT_ should be changed to PROP_

This requires more than a single day to make, and may cause big
instability. More than 1000 lines should be changed.

What would be the best solution?
- don't rename 'status' to 'property',even though 'status' is misleading
- change 'status' to 'property' only in a small part of the code
  which may cause naming incoherency
- make the changes gradually
- postpone and do it later with a cleaner code
- don't make setter / getter methods
...

I think, that this kind of immense changes requires some discussion.
