1. unifying the two status types 
 
Q_UINT64 is a predefined QT type for that reason.  
By the C standards, sizeof( long long ) >= sizeof( long ) 
 
So the long long status variable is not very safe. I think  
using the QT version is safer. 
 
But on the most unix systems  
INT64 and UINT64 are supported. I think, it's <sys/types.h> 
 
2. faster access for the status variables 
 
I think, that this can be solved easily by a simple inline declaration 
 
inline status_type status() { return status_; } 
 
I think, that the most compiler will make the inline optimization... 
 
I think, that the speed is only important for the getter methods. 
The setter method can be slow by emitting the change signal. 
 
3. the status() <-> properties() change is acceptable for me 
 
We cannot add the properties into getSubType() as the length of the note 
is stored in it. The properties name is very informative.
