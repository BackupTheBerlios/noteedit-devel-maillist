Hi Matevz,

Enums for me are just what they are: enumberations, so
it is especially useful if you have comboboxes in the
user interfaces and replace their number with enum.

If the entry of comboboxes change under a certain
context, you can even use a simple mapping to map
the used entry to the real enum number using a
vector:

typedef enum
{
  Entry_First    = 0,  // Not necessary, just for easier identification
  Entry_Special,
  Entry_All,
  Entry_Notes,
  Entry_Usual,
  Entry_Tea,
  Entry_Coffee,
  Entry_NeverUsed,
  Entry_Last
} Entry_T;

int entryMap[10];
entryMap[0] = Entry_First
entryMap[1] = Entry_All
entryMap[2] = Entry_NeverUsed
entryMap[3] = Entry_Tea
entryMap[4] = Entry_Last

This way you can easily fill your combobox with
the entries until Entry_Last is found (or start
with the last entry until the first).

In your case it depends how you use the mode.
If several modes can be used at the same time,
I wouldn't normally use enums except you define
them in a way so you can bitwise and / or them:

typedef enum
{
  NEM_Select = 1,
  NEM_Insert = 2,
  NEM_Edit   = 4
   .
   .
   .
}
NoteEditMode_T;
