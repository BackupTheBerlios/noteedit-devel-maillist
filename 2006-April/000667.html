Hi,

This is really simple to answer: A old bug in the Qt designer.
If you add custom widgets to an ui file using the designer he
adds tons of include hints (for every widget based on the custom
widget one extra) using the class name in lower case letters.
NScaleEdit is part of a completely different file named scaleEd.ui
so the correct include hint would be scaleEd.h. Unfortunately somewhen
the Trolls did make a change in their include hint handling and now
add somewhere an include command, if the ui file is compiled with the
uic. If you still use Qt3.3.3 this does not happen.
